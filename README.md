# 我的世界超多联史莱姆区块寻找计划 #
### 目标 ###

  我的世界一共有2^64个种子，其中史莱姆区块的分布只取决与种子的后48位，也就是说一共有2^48种可能的分布。
  
  为了方便建造和使用，程序只会计算种子的1万格范围内所有可能的挂机点。

  挂机点默认在区块中心的y-64坐标，所以外围的24-128范围均是可刷怪范围。
  
### 使用方法 ###

  首先确保你有一张N卡。我的程序可能对4060有一些特别的优化，不确定程序是否能在20系显卡上正常运行。
  
  *如果你没有N卡，你可以使用我筛出来的[结果](https://github.com/ldqm1/SlimeFarmLocator/tree/main/check_single_seed/15000_images)，或使用[rust版本](https://github.com/ldqm1/SlimeFarmLocator/tree/main/findslime_rust)。*

  在命令行输入nvcc，如果找不到命令，则去下载Nvidia的驱动程序并添加环境变量。
  
  下载[程序](https://github.com/ldqm1/SlimeFarmLocator/blob/main/findslime_cuda/main7.cu),使用编译命令 
  
  ``` nvcc -O3 -use_fast_math  -Xcompiler "/O2 /arch:AVX2"  -maxrregcount=64 -o findslime.exe main7.cu ```
  
  进行编译。
  
  然后在命令控制台运行
  
  ```findslime.exe <起始种子> <运行次数> ```
  
  随后会在此目录生成`result.csv`文件，可以将此文件放入findslime目录并使用`python data.py`来把csv文件转为sqlite数据库，运行结束会自动删除csv文件。

  当你已经获取到一些史莱姆区块较多的种子，你可以将种子的列表放入`list.txt`,然后将文件放入[check_single_seed](https://github.com/ldqm1/SlimeFarmLocator/tree/main/check_single_seed),并运行start.bat,程序会将有效方块面积大于14500的种子和大于15000的种子放在check_single_seed目录下。

  这部分程序使用了这个项目[slimefinder](https://github.com/Nukelawe/slimefinder),我添加了一个python脚本使它可以批量计算。

  图片的名字格式为`有效方块数_种子_中心坐标.png`。

  另外，由于史莱姆区块分布只取决于低48位，所以可以通过将种子加上2^48或减去2^48来获取一共32767个相同分布史莱姆区块的种子以避免那个位置生成幽暗之域或导入其他筛种器加其他条件继续筛种。
